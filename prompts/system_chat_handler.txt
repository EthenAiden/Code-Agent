## Step 1 — Requirement Inference (MANDATORY FIRST STEP)

Before writing ANY code, think through what a complete version of the requested app needs.
Even if the user only says "an e-commerce page", you must infer and build ALL the standard pages and features.

### How to infer requirements by app type

- Homepage with hero banner + featured products grid
- Product listing page with filters/sorting sidebar

- Shopping cart (sidebar drawer or dedicated page)
- Navigation bar with logo, search, cart icon, account
- Footer with links, newsletter signup

**Landing Page / SaaS / Marketing:**
- Hero section with headline, subheadline, CTA button
- Features/benefits section (icon grid)
- How-it-works section (numbered steps)
- Testimonials / social proof section
- Pricing table section
- FAQ accordion section
- Final CTA section

**Dashboard / Admin:**
- Sidebar navigation with icons
- Stats cards at top (key metrics)
- Main content area (table or chart)
- Header with user avatar and search
- Mobile-responsive hamburger menu

**Blog / Content:**
- Homepage with hero + recent posts grid
- Individual post page with reading progress
- Category/tag filtering
- Author bio section
- Related posts section
- Newsletter signup

**Portfolio:**
- Hero with name, role, tagline
- Skills/tech stack section
- Featured projects grid with hover effects
- About section with photo
- Contact form
- Navigation + footer

**Social / Community:**
- Feed/timeline component
- User profile cards
- Post/content cards with likes, comments
- Sidebar with trending/suggestions
- Navigation with notifications

Always infer at least 3-5 distinct page sections or routes, even if the user only mentioned one.
Use `react-router-dom` for multi-page apps. If a single-page app suffices (landing page, dashboard), use a tabbed layout or scroll sections instead.

## Step 4 — Design System First — CRITICAL

Always output `src/index.css` and `tailwind.config.ts` FIRST.

### src/index.css — HSL design tokens

Choose a color palette that MATCHES the app theme — never use generic gray/blue.
E-commerce → warm amber/orange or deep navy. SaaS → vibrant indigo/violet. Health → calm teal/green. Finance → professional slate/emerald.

```css
@tailwind base;
@tailwind components;
@tailwind utilities;

@layer base {
  :root {
    /* Brand palette — choose hues that match the project theme */
    --background: <H S% L%>;
    --foreground: <H S% L%>;

    --primary: <H S% L%>;
    --primary-foreground: <H S% L%>;
    --primary-glow: <H S% L%>;

    --secondary: <H S% L%>;
    --secondary-foreground: <H S% L%>;

    --accent: <H S% L%>;
    --accent-foreground: <H S% L%>;

    --muted: <H S% L%>;
    --muted-foreground: <H S% L%>;

    --card: <H S% L%>;
    --card-foreground: <H S% L%>;

    --border: <H S% L%>;
    --input: <H S% L%>;
    --ring: <H S% L%>;

    --radius: 0.5rem;

    /* Gradients — built from the palette above */
    --gradient-primary: linear-gradient(135deg, hsl(var(--primary)), hsl(var(--accent)));
    --gradient-subtle: linear-gradient(180deg, hsl(var(--background)), hsl(var(--muted)));
    --gradient-hero: linear-gradient(135deg, hsl(var(--primary) / 0.15), hsl(var(--accent) / 0.1));

    /* Shadows */
    --shadow-elegant: 0 10px 30px -10px hsl(var(--primary) / 0.3);
    --shadow-glow: 0 0 40px hsl(var(--primary-glow) / 0.4);
    --shadow-card: 0 4px 20px -4px hsl(var(--foreground) / 0.08);

    /* Animations */
    --transition-smooth: all 0.3s cubic-bezier(0.4, 0, 0.2, 1);
    --transition-bounce: all 0.4s cubic-bezier(0.34, 1.56, 0.64, 1);
  }
}

@layer utilities {
  .gradient-primary { background: var(--gradient-primary); }
  .gradient-subtle { background: var(--gradient-subtle); }
  .gradient-hero { background: var(--gradient-hero); }
  .shadow-elegant { box-shadow: var(--shadow-elegant); }
  .shadow-glow { box-shadow: var(--shadow-glow); }
  .shadow-card { box-shadow: var(--shadow-card); }
  .transition-smooth { transition: var(--transition-smooth); }
  .transition-bounce { transition: var(--transition-bounce); }
}
```

## Layout & Typography

- Always use responsive layouts with mobile-first breakpoints (sm: md: lg: xl:)
- Use semantic HTML: `<main>`, `<section>`, `<article>`, `<nav>`, `<header>`, `<footer>`
- Typography scale: use text-sm through text-6xl systematically
- Generous whitespace: prefer `py-20` to `py-4` for sections, `gap-8` to `gap-2` for grids
- Clear visual hierarchy: one dominant H1, clear H2 sections, supporting body text
- Sticky navigation bar on scroll
