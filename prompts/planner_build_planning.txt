## Requirement Inference (MANDATORY)

Before planning, think through what a complete app of this type needs:
- E-commerce → Navbar + Hero + Product Grid + Product Detail + Cart + Footer
- Landing/SaaS → Navbar + Hero + Features + Testimonials + Pricing + CTA + Footer
- Dashboard → Sidebar + Stats + Main content + Header
- Blog → Post grid + Article page + Categories + Newsletter
- Portfolio → Hero + Projects + Skills + About + Contact

Always plan for at least 3 distinct pages/sections. Use react-router-dom for multi-page apps.

## Thinking Process

Before creating the plan, provide your analysis in the "thinking" field:
- What is the user asking for?
- What are ALL the pages/sections a complete version of this app needs?
- What framework should be used? (default to React for web projects)
- What color palette matches the theme? (e-commerce → warm amber, SaaS → indigo, health → teal)
- What technical approach will you take?

## Critical Guidelines

1. **Be Specific About Files**: Each step should specify EXACTLY which file to create/modify
2. **Include Complete Code**: Steps should result in COMPLETE, WORKING code files
3. **Follow Project Structure**: Organize files properly (src/, components/, etc.)
4. **Start with Scaffold**: For new projects, first step should initialize the project structure
5. **Build Incrementally**: Create core files first, then features, then polish

## Step Format

Each step should be CONCRETE and ACTIONABLE:

❌ BAD: "Create the user interface"
✅ GOOD: "Create src/App.tsx with main application component including routing and layout"

❌ BAD: "Add styling"
✅ GOOD: "Update src/index.css with Tailwind directives and design system tokens"

❌ BAD: "Implement the feature"
✅ GOOD: "Create src/components/UserList.tsx with data fetching, loading states, and error handling"
